# Notifications Overview

Send email, SMS, and in-app notifications to staff and customers.

## Overview

The notifications system allows you to communicate with staff and customers through multiple channels:

- **Email**: Rich content via SendGrid
- **SMS**: Text messages via Twilio
- **Push**: Mobile app notifications
- **In-App**: Real-time dashboard notifications

## Accessing Notifications

Navigate to **Notifications** in the sidebar to access the notification center.

## Notification Channels

### Email

Best for:
- Detailed information
- Order confirmations
- Newsletters and marketing
- Rich HTML content

**Requirements:**
- SendGrid API key configured
- Verified sender domain

### SMS

Best for:
- Urgent alerts
- Short updates
- Time-sensitive information
- Queue notifications

**Requirements:**
- Twilio account credentials
- Phone numbers in E.164 format (+1234567890)

**Note:** Messages over 160 characters may be split into multiple SMS.

### Push Notifications

Best for:
- Real-time alerts
- App engagement
- Staff shift reminders

**Requirements:**
- Mobile app installed
- Push notifications enabled

### In-App Notifications

Best for:
- Dashboard alerts
- System messages
- Non-urgent updates

**Note:** Always available, no external configuration needed.

## Best Practices

### Choosing the Right Channel

- **Urgent**: Use SMS for time-sensitive information
- **Detailed**: Use email for complex content
- **Quick updates**: Use push or in-app
- **Marketing**: Use email with unsubscribe options

### Message Content

- Keep SMS under 160 characters when possible
- Use clear subject lines for email
- Include relevant details (dates, times, order numbers)
- Personalize with template variables

### Timing

- Avoid sending at odd hours
- Consider timezone differences
- Space out marketing messages
- Send confirmations immediately
