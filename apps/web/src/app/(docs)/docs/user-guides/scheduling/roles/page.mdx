# Schedule Roles

View and manage roles used for organizing shifts on the schedule.

## Overview

Schedule roles are categories that organize shifts by function or position. Each role has a unique color that makes it easy to identify different shift types on the calendar view.

## Accessing Roles

1. Navigate to **Schedule** in the sidebar
2. Click **Roles**

## Role List

The roles page displays all configured roles:

| Column | Description |
|--------|-------------|
| Role | Name with color indicator |
| Color | Hex color code |
| Description | Role description |
| Status | Active or Inactive |
| Sort Order | Display priority |

## Role Summary Cards

At the top of the page, summary cards show:

| Card | Description |
|------|-------------|
| Total Roles | All configured roles |
| Active Roles | Roles available for scheduling |
| Inactive Roles | Disabled roles |

## Role Properties

Each role has the following properties:

| Property | Description |
|----------|-------------|
| Name | Role display name |
| Color | Visual identifier (hex color) |
| Description | What this role does |
| Active Status | Whether role can be used |
| Sort Order | Position in lists |

## Color Coding

Colors help identify roles at a glance:

### On the Calendar

- Shifts display in their role's color
- Easy visual identification
- Quick coverage assessment
- Pattern recognition

### Color Legend

The page includes a color legend showing:
- All active roles
- Their assigned colors
- Visual reference for calendar

## Using Roles

### When Creating Shifts

1. Create a new shift
2. Select a role from the dropdown
3. Role color applies automatically
4. Shift appears on calendar with that color

### When Creating Templates

1. Create a new template
2. Assign a role to the template
3. Generated shifts inherit the role

### For Filtering

- Filter shifts by role
- View only specific role types
- Analyze coverage by role

## Active vs Inactive Roles

### Active Roles

- Available for new shifts
- Appear in role dropdowns
- Show in color legend
- Used for templates

### Inactive Roles

- Cannot be assigned to new shifts
- Hidden from dropdowns
- Existing shifts keep the role
- Preserved for historical data

## Sort Order

Sort order controls role display:
- Lower numbers appear first
- Used in dropdowns and lists
- Organize by priority or frequency
- Adjust as needs change

**Example:**
| Role | Sort Order | Position |
|------|------------|----------|
| Actor | 0 | First |
| Security | 1 | Second |
| Box Office | 2 | Third |
| Misc | 99 | Last |

## Color Best Practices

### Choosing Colors

- Use distinct, contrasting colors
- Avoid similar shades for different roles
- Consider colorblind accessibility
- Test on calendar view

### Color Consistency

- Maintain same colors over time
- Document color assignments
- Train staff on color meanings

### Recommended Palette

| Category | Suggested Colors |
|----------|------------------|
| Performance | Purple, Magenta |
| Operations | Blue, Cyan |
| Sales | Green, Teal |
| Security | Red, Orange |
| Support | Yellow, Gray |

## Role Planning

### Determining Roles

Consider your needs:
- What positions exist?
- How many distinct roles?
- Any overlap between roles?
- Future role needs?

### Role Granularity

Balance between:
- Too few: Hard to distinguish
- Too many: Overwhelming
- Right amount: Clear organization

## Troubleshooting

### Role Not Appearing

- Check if role is active
- Verify attraction assignment
- Refresh the page
- Check permissions

### Cannot Assign Role

- Ensure role is active
- Check shift is editable
- Verify role exists
- Contact admin

### Colors Not Showing

- Check browser compatibility
- Refresh the page
- Verify color is set
- Clear browser cache

### Wrong Sort Order

- Check sort order values
- Lower numbers appear first
- Update and save
- Refresh to see changes
